Proof of Concept: Specifications Display Rules & Logic

Concept Overview

This proof of concept demonstrates how to display aggregated vehicle data across all variants of a model. Rather than showing individual trim levels separately, the page presents a unified view with ranges and options, helping users understand the full scope of what's available for a vehicle model.

Aggregation Rules

  • Price Range: Display minimum to maximum across all variants. If all variants have the same price, show a single value.
  • Specifications: Collect all unique values from variants, sort them logically, and present with "or" separators
  • Multiple Values: Join with commas, placing "or" before the last item (e.g., "A, B, or C")
  • Two Values: Simply join with "or" (e.g., "A or B")
  • Single Value: Display without separators

Transmission Display Rules

  • Electric Vehicles: When transmission has 1 speed and vehicle is electric â†’ display as "Single-Speed Auto"
  • CVT: When transmission speeds = "Variable" â†’ display as "Auto (CVT)"
  • DCT: When transmission description contains "dual clutch" or "DCT" â†’ display as "[X]-speed Auto (DCT)"
  • Standard Automatic: Display as "[X]-speed auto" (lowercase "auto")
  • Manual: Display as "[X]-speed manual"
  • Educational Tooltips: CVT, DCT, and Single-Speed Auto include hover tooltips explaining the technology to users

Fuel Type Display Rules

Fuel type displays the actual fuel(s) the vehicle uses, sourced from fuelType and fuel.fuelOtherFuelType:

  • Plug-in Hybrid Vehicles: Display both fuel types as "[Primary]/[Secondary]" (e.g., "Electric/Premium Unleaded")
  • Hybrid Vehicles: Display both fuel types as "[Primary]/[Secondary]" if fuelOtherFuelType is valid
  • Pure Electric/Combustion: Display only the primary fuelType (e.g., "Electric", "Unleaded", "Diesel")
  • Data Validation: Filter out "None" values from fuelOtherFuelType to prevent displaying invalid data
  • Sort Order: When multiple fuel types exist across variants: Petrol â†’ Unleaded â†’ Diesel â†’ LPG â†’ Hybrid â†’ Plug-in Hybrid â†’ Electric
  • Powertrain vs Fuel: This field shows what you put in the tank. See "Powertrain Type" for vehicle classification (PHEV, HEV, etc.)

Powertrain Type Display Rules

The Powertrain Type field classifies the vehicle's propulsion system. This is distinct from Fuel Type:

  • Battery Electric Vehicle â†’ "Electric"
  • Hybrid Electric Vehicle â†’ "Hybrid"
  • Plug-in Hybrid Electric Vehicle / Plug-In Hybrid Electric Vehicle â†’ "PHEV"
  • Combustion â†’ "Combustion"
  • Case Handling: Supports both "Plug-in" and "Plug-In" (capital I) variants for maximum data compatibility
  • Key Difference: Powertrain Type = vehicle classification; Fuel Type = what fuels the vehicle
  • Other values display as-is

UI Interaction Patterns

  • Vehicle Selector: Dropdown allows switching between models to preview different data sets in the POC
  • Read More Toggle: Expands description and temporarily hides specifications to reduce visual clutter
  • Image Navigation: Arrow buttons cycle through available vehicle images
  • Transmission Tooltips: Hover state on CVT, DCT, and Single-Speed Auto provides educational context
  • Tooltip Positioning: Dynamically adjusts to stay within viewport boundaries
  • Dropdown Closure: Automatically closes when clicking outside the selector area

Price Display Rules

  • Format: Australian dollar format with no decimal places (e.g., "$52,990")
  • Single Price: Display as "$XX,XXX" when all variants have the same price
  • Price Range: Display as "$XX,XXX - $XX,XXX" when prices vary across variants
  • No Price: Display "Price TBA" when price data is unavailable or zero
  • Locale: Uses Australian number formatting (en-AU)

Vehicle Naming Rules

  • Page Title (H1): Use displayNameFull for the main heading (e.g., "Ford Ranger")
  • Breadcrumb Make: Use make field for the manufacturer breadcrumb
  • Breadcrumb Model: Use displayNameShort for the model breadcrumb, fallback to model
  • Fallback: If displayNameFull is missing, construct from make + model
  • Data Source: Name fields are extracted from the first vehicle variant in the dataset

Note: This is a proof of concept demonstrating display rules and aggregation logic. Production implementation will use different data sources and loading mechanisms, but should maintain these formatting and aggregation rules for consistency across the platform.
