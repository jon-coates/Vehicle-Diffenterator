How This Works

Overview

This vehicle hero page dynamically loads and displays vehicle data from JSON files. It aggregates information across all variants of a vehicle model to provide a comprehensive overview including price ranges, available transmissions, and specifications.

Data Loading

  • Vehicle data is fetched from vehicleData/ directory
  • Each JSON file contains an array of vehicle variants
  • Data structure supports data.vehicles, data.data, or direct array format
  • The first variant determines the vehicle name and make/model information

Aggregation Rules

  • Price: Displays minimum to maximum across all variants. Shows single price if all variants are the same.
  • Transmission: Collects unique transmission types and formats them with "or" separator
  • Fuel Type: Formatted based on powertrain type, then sorted logically (Petrol â†’ Diesel â†’ Hybrid â†’ Plug-in Hybrid â†’ Electric)
  • Powertrain: Abbreviated display names (Electric, Hybrid, PHEV, Combustion)
  • Specifications: All unique values from variants are collected, sorted, and displayed with proper formatting
  • Body Types: Multiple body types are joined with commas and "or" before the last item

Transmission Formatting

  • Electric Vehicles: Single-speed transmissions display as "Single-Speed Auto"
  • CVT: Variable-speed transmissions display as "Auto (CVT)" with tooltip
  • DCT: Dual-clutch transmissions show speed count + "Auto (DCT)" with tooltip
  • Standard: Format as "[X]-speed auto" or "[X]-speed manual"
  • Tooltips provide educational context for special transmission types

Fuel Type Display

Fuel types are intelligently formatted based on powertrain (user-friendly full names):

  • Electric vehicles: Display as "Electric"
  • Hybrid vehicles: Display as "Hybrid"
  • Plug-in hybrids: Display as "Plug-in Hybrid" (full name)
  • Combustion vehicles: Show actual fuel type (Petrol, Diesel, Unleaded, LPG)
  • Multiple fuel types are sorted logically: Petrol â†’ Diesel â†’ LPG â†’ Hybrid â†’ Plug-in Hybrid â†’ Electric

Powertrain Display

Powertrain types use abbreviated names for brevity:

  • Battery Electric Vehicle â†’ "Electric"
  • Hybrid Electric Vehicle â†’ "Hybrid"
  • Plug-in Hybrid Electric Vehicle â†’ "PHEV" (abbreviated)
  • Combustion â†’ "Combustion"

UI Interactions

  • Vehicle Selector: Dropdown next to title allows switching between available vehicles
  • Read More: Expands description and temporarily hides specifications grid
  • Image Navigation: Left/right arrows cycle through available images
  • Tooltips: Hover over transmission values with special types to see explanations
  • Dropdown: Closes automatically when clicking outside the selector area

Data Requirements

Required Fields: Each vehicle variant should include:

  • make, model - Basic identification
  • modelDetails.displayNameFull - Full display name for H1
  • modelDetails.displayNameShort - Short name for breadcrumb
  • price - Retail price (0 = TBA)
  • transmissionType, transmissionNumberOfSpeeds - Transmission details
  • transmissionDescription - Optional, used to detect DCT
  • drivenWheels - Drive configuration (FWD, RWD, AWD, 4x4)
  • fuelType - Raw fuel type (used for combustion vehicles)
  • powertrainType - Critical for fuel type and powertrain display
  • bodyType - Vehicle body style

Technical Notes

  • All global functions use async/await for data loading
  • Price formatting uses Australian locale (en-AU)
  • Tooltip positioning dynamically adjusts to viewport boundaries
  • Event delegation prevents memory leaks when updating transmission tooltips
  • Selected vehicle in dropdown is marked with .selected class
  • Default vehicle (Ford Ranger) loads on page initialisation

Adding New Vehicles: Place JSON file in vehicleData/ directory and add a corresponding dropdown item in the HTML with the correct data-file attribute and onclick handler.
