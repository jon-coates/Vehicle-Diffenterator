Aggregation Rules

  • Price Range: Display minimum to maximum across all variants. If all variants have the same price, show a single value.
  • Specifications: Collect all unique values from variants, sort them logically, and present with "or" separators
  • Multiple Values: Join with commas, placing "or" before the last item (e.g., "A, B, or C")
  • Two Values: Simply join with "or" (e.g., "A or B")
  • Single Value: Display without separators

Price Display

  • Format: Australian dollar format with no decimal places (e.g., "$52,990")
  • Single Price: Display as "$XX,XXX" when all variants have the same price
  • Price Range: Display as "$XX,XXX - $XX,XXX" when prices vary across variants
  • No Price: Display "Price TBA" when price data is unavailable or zero
  • Locale: Uses Australian number formatting (en-AU)

Fuel Type Display

Fuel type displays the actual fuel(s) the vehicle uses, sourced from fuelType and fuel.fuelOtherFuelType:

  • Plug-in Hybrid Vehicles: Display both fuel types as "[Primary]/[Secondary]" (e.g., "Electric/Premium Unleaded")
  • Hybrid Vehicles: Display both fuel types as "[Primary]/[Secondary]" if fuelOtherFuelType is valid
  • Pure Electric/Combustion: Display only the primary fuelType (e.g., "Electric", "Unleaded", "Diesel")
  • Data Validation: Filter out "None" values from fuelOtherFuelType to prevent displaying invalid data
  • Sort Order: When multiple fuel types exist across variants: Petrol â†’ Unleaded â†’ Diesel â†’ LPG â†’ Hybrid â†’ Plug-in Hybrid â†’ Electric

Powertrain Type Display

The Powertrain Type field classifies the vehicle's propulsion system. This is distinct from Fuel Type:

  • Battery Electric Vehicle â†’ "Electric"
  • Hybrid Electric Vehicle â†’ "Hybrid"
  • Plug-in Hybrid Electric Vehicle / Plug-In Hybrid Electric Vehicle â†’ "PHEV"
  • Combustion â†’ "Combustion"
  • Case Handling: Supports both "Plug-in" and "Plug-In" (capital I) variants for data compatibility
  • Key Difference: Powertrain Type = vehicle classification; Fuel Type = what fuels the vehicle

Vehicle Naming

  • Vehicle Title: Use displayNameFull for the main heading (e.g., "Ford Ranger")
  • Fallback: If displayNameFull is missing, construct from make + model
  • Data Source: Name fields are extracted from the first vehicle variant in the dataset
